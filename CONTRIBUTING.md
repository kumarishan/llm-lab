# Contributing to LLM Lab

Thanks for helping improve these tutorials. This repo is educational material derived from upstream projects (for example [microgpt](https://karpathy.github.io/2026/02/12/microgpt/) and [nanochat](https://github.com/karpathy/nanochat)); contributions should stay accurate, kind, and easy to follow.

## Start with an issue

**Please open an issue before you invest in a large change or new tutorial series.** That gives space to:

- **Improve** — typos, broken links, unclear explanations, better exercises, or chapter ordering.
- **Vet** — technical accuracy, alignment with the source code or upstream docs, and pedagogy (too fast, too slow, missing prerequisites).
- **Plan** — agree on scope so a pull request matches what maintainers and readers need.

Small fixes (single typo, obvious broken link) can go straight to a PR if you prefer, but an issue is still welcome for tracking.

## Suggesting other repositories for tutorials

If you know a **repo that would benefit from a chapter-by-chapter tutorial** in this style (similar to what the [repo-tutorial skill](https://github.com/kumarishan/kistack) produces), open an issue and include:

- Link to the repository (and license, if not obvious).
- Why it is a good learning target (audience, uniqueness, size of codebase).
- Whether you are offering to help write or review chapters.

We cannot promise every suggestion will be taken on, but discussion in issues helps prioritize and vet ideas.

## Pull requests

- Reference the related issue when one exists.
- Keep changes focused (one logical improvement per PR when possible).
- Match the tone and structure of existing chapters: clear headings, hands-on steps where appropriate, and accurate references to file and line context when citing code.

## Code of conduct

Be respectful in issues and pull requests. Assume good intent and prefer constructive feedback.
